## When to Archive

Documents should be archived when they:
- Document completed one-time activities (migrations, refactoring)
- Are session-specific logs
- Have been superseded by newer documentation
- Contain historical decisions that inform future work

## When NOT to Archive

Keep documents in active locations when they:
- Describe ongoing processes or procedures
- Are actively referenced by users or developers
- Contain configuration that may need updates
