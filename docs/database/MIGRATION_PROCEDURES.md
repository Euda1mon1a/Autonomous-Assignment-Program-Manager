***REMOVED******REMOVED*** Overview

Migrations manage database schema changes using Alembic. They:
- Track database evolution
- Enable rollback capability
- Maintain production safety
- Provide version control for schema

***REMOVED******REMOVED******REMOVED*** Key Principles

1. **One migration per logical change**: Don't combine unrelated changes
2. **Backward compatibility**: Old code should work during gradual rollout
3. **Reviewable migrations**: Keep them readable for code review
4. **Tested rollbacks**: Always test `downgrade` works
5. **Data safety**: Plan for data migrations carefully

***REMOVED******REMOVED******REMOVED*** Architecture

```
Models (SQLAlchemy)
    ↓
Migrations (Alembic)
    ↓
Database Schema
```

***REMOVED******REMOVED*** Migration Types

***REMOVED******REMOVED******REMOVED*** 1. Schema Migrations (Autogenerated)

Structural changes detected and generated automatically.

**Examples**:
- Add/remove columns
- Add/remove tables
- Change column types
- Add/remove constraints
- Add/remove indexes

**Generation**:
```bash
cd backend
alembic revision --autogenerate -m "Add email column to persons"
```

***REMOVED******REMOVED******REMOVED*** 2. Data Migrations (Manual)

Application-level data transformations.

**Examples**:
- Populate new columns from existing data
- Rename values in enum columns
- Merge/split columns
- Denormalize for performance

**Template**:
```python
"""Data migration: Rename role values."""

from alembic import op
import sqlalchemy as sa


def upgrade() -> None:
    """Upgrade: Rename OLD_ROLE to NEW_ROLE."""
    op.execute(
        "UPDATE persons SET role = 'NEW_ROLE' WHERE role = 'OLD_ROLE'"
    )


def downgrade() -> None:
    """Downgrade: Restore original role values."""
    op.execute(
        "UPDATE persons SET role = 'OLD_ROLE' WHERE role = 'NEW_ROLE'"
    )
```

***REMOVED******REMOVED******REMOVED*** 3. Complex Migrations

Multi-step migrations requiring orchestration.

**Example**: Add new table with foreign keys from existing table

```python
"""Add certifications table."""

from alembic import op
import sqlalchemy as sa


def upgrade() -> None:
    """Create certifications table with proper constraints."""
    ***REMOVED*** Step 1: Create table
    op.create_table(
        'certifications',
        sa.Column('id', sa.String(36), primary_key=True),
        sa.Column('person_id', sa.String(36), sa.ForeignKey('persons.id')),
        sa.Column('name', sa.String(255), nullable=False),
        sa.Column('expires_at', sa.DateTime),
    )

    ***REMOVED*** Step 2: Create index
    op.create_index('ix_certifications_person_id', 'certifications', ['person_id'])

    ***REMOVED*** Step 3: Add constraint
    op.create_unique_constraint(
        'uq_certifications_person_name',
        'certifications',
        ['person_id', 'name']
    )


def downgrade() -> None:
    """Remove certifications table."""
    op.drop_table('certifications')
```

***REMOVED******REMOVED*** Creating Migrations

***REMOVED******REMOVED******REMOVED*** Step 1: Modify Model

Edit the SQLAlchemy model in `backend/app/models/`:

```python
***REMOVED*** backend/app/models/person.py
from sqlalchemy import Column, String, DateTime

class Person(Base):
    __tablename__ = "persons"

    id = Column(String(36), primary_key=True)
    name = Column(String(255), nullable=False)
    email = Column(String(255))  ***REMOVED*** NEW COLUMN
    created_at = Column(DateTime, default=datetime.utcnow)
```

***REMOVED******REMOVED******REMOVED*** Step 2: Generate Migration

```bash
cd backend
alembic revision --autogenerate -m "Add email column to persons"
```

This creates `alembic/versions/abc123_add_email_column_to_persons.py`

***REMOVED******REMOVED******REMOVED*** Step 3: Review Migration

**ALWAYS** review autogenerated migrations - they're not perfect!

```python
***REMOVED*** alembic/versions/abc123_add_email_column_to_persons.py
def upgrade() -> None:
    op.add_column('persons', sa.Column('email', sa.String(255), nullable=True))


def downgrade() -> None:
    op.drop_column('persons', 'email')
```

**Check for**:
- Correctness of column definitions
- Missing constraints
- Nullable vs NOT NULL
- Indexes needed
- Foreign keys
- Unique constraints

***REMOVED******REMOVED******REMOVED*** Step 4: Edit if Needed

```python
***REMOVED*** Enhanced version
def upgrade() -> None:
    ***REMOVED*** Add column with constraint
    op.add_column(
        'persons',
        sa.Column('email', sa.String(255), nullable=False, unique=True)
    )

    ***REMOVED*** Create index for frequent queries
    op.create_index('ix_persons_email', 'persons', ['email'])


def downgrade() -> None:
    op.drop_index('ix_persons_email')
    op.drop_column('persons', 'email')
```

***REMOVED******REMOVED******REMOVED*** Step 5: Test Locally

```bash
***REMOVED*** Apply migration
alembic upgrade head

***REMOVED*** Verify schema
python -c "from app.db.session import engine; \
from app.db.base import Base; \
from app.models import *; \

***REMOVED*** Test rollback

```


```python
***REMOVED*** test_migration.py
from sqlalchemy import inspect, create_engine
from app.db.base import Base

def test_migration():
    """Test migration applied correctly."""
    engine = create_engine("postgresql://...")
    inspector = inspect(engine)

    ***REMOVED*** Check table exists
    assert 'persons' in inspector.get_table_names()

    ***REMOVED*** Check column exists
    columns = [c['name'] for c in inspector.get_columns('persons')]
    assert 'email' in columns

    ***REMOVED*** Check column type
    email_col = next(c for c in inspector.get_columns('persons')
                     if c['name'] == 'email')
    assert email_col['type'].__class__.__name__ == 'String'

    print("✓ Migration validation passed")

if __name__ == "__main__":
    test_migration()
```

***REMOVED******REMOVED*** Common Patterns

***REMOVED******REMOVED******REMOVED*** Pattern 1: Add New Column with Default

```python
def upgrade() -> None:
    """Add status column with default."""
    op.add_column(
        'assignments',
        sa.Column(
            'status',
            sa.String(50),
            nullable=False,
            server_default='ACTIVE'
        )
    )


def downgrade() -> None:
    op.drop_column('assignments', 'status')
```

***REMOVED******REMOVED******REMOVED*** Pattern 2: Rename Column

```python
def upgrade() -> None:
    """Rename created_at to created_timestamp."""
    op.alter_column(
        'persons',
        'created_at',
        new_column_name='created_timestamp'
    )


def downgrade() -> None:
    op.alter_column(
        'persons',
        'created_timestamp',
        new_column_name='created_at'
    )
```

***REMOVED******REMOVED******REMOVED*** Pattern 3: Add Foreign Key

```python
def upgrade() -> None:
    """Add foreign key to assignments."""
    op.create_foreign_key(
        'fk_assignments_person_id',
        'assignments',
        'persons',
        ['person_id'],
        ['id']
    )


def downgrade() -> None:
    op.drop_constraint(
        'fk_assignments_person_id',
        'assignments',
        type_='foreignkey'
    )
```

***REMOVED******REMOVED******REMOVED*** Pattern 4: Create Index

```python
def upgrade() -> None:
    """Create index for performance."""
    op.create_index(
        'ix_assignments_person_date',
        'assignments',
        ['person_id', 'date']
    )


def downgrade() -> None:
    op.drop_index('ix_assignments_person_date')
```

***REMOVED******REMOVED******REMOVED*** Pattern 5: Data Migration with Condition

```python
def upgrade() -> None:
    """Populate new column from existing data."""
    op.execute(
        """
        UPDATE persons
        SET employment_type = 'FULL_TIME'
        WHERE is_active = TRUE
        """
    )


def downgrade() -> None:
    op.execute(
        """
        UPDATE persons
        SET employment_type = NULL
        """
    )
```

***REMOVED******REMOVED*** Troubleshooting

***REMOVED******REMOVED******REMOVED*** Issue: "No such table" errors

**Cause**: Running app code before migrations applied

**Solution**:
```bash
alembic upgrade head
python app/main.py
```

***REMOVED******REMOVED******REMOVED*** Issue: "Foreign key constraint fails"

**Cause**: Dropping table while other tables reference it

**Solution**:
```python
***REMOVED*** Drop dependent tables first
def upgrade():
    op.drop_table('assignments')
    op.drop_table('persons')
```

***REMOVED******REMOVED******REMOVED*** Issue: Alembic can't find changes

**Cause**: SQLAlchemy model not imported properly

**Solution**:
1. Check imports in `alembic/env.py`
2. Ensure all models imported in `app/models/__init__.py`
3. Verify `target_metadata` is correct

```python
***REMOVED*** alembic/env.py
from app.db.base import Base
from app.models import *  ***REMOVED*** Import all models

target_metadata = Base.metadata
```

***REMOVED******REMOVED******REMOVED*** Issue: "Current database version has no target revision"

**Cause**: Database not initialized

**Solution**:
```bash
alembic upgrade head
```

***REMOVED******REMOVED******REMOVED*** Issue: Migrations keep failing in test

**Cause**: Test database not properly isolated

**Solution**:
```python
***REMOVED*** conftest.py
@pytest.fixture
def db():
    """Fresh database for each test."""
    Base.metadata.create_all(engine)
    yield SessionLocal()
    Base.metadata.drop_all(engine)
```

```yaml
          cd backend
          alembic upgrade head

      - name: Test downgrade
        env:
          DATABASE_URL: postgresql://postgres:postgres@localhost/test_db
        run: |
          cd backend
          alembic downgrade base

      - name: Show history

          cd backend
```
