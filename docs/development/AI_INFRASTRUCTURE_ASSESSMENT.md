## 1. What This Repo Is (Context for All Agents)

**Residency Scheduler** — TAMC (Tripler Army Medical Center) military medical residency scheduling with ACGME compliance.

- **Backend**: FastAPI + SQLAlchemy 2.0 + PostgreSQL + Celery/Redis
- **Frontend**: Next.js 14, React 18, TailwindCSS — **58 pages** across 15 user routes + 32 admin routes
- **Solvers**: Google OR-Tools CP-SAT (block scheduling) + ARO CP-SAT (annual rotation planning)
- **Export**: XLSX via `TAMCBlockExporter` (the old XML/Rosetta bridge is fully deleted)
- **MCP Server**: 74+ AI tools on `http://127.0.0.1:8080/mcp`
- **Codebase quality**: Mypy zero errors, ~8,300 tests (some known failures in batch/resilience), ESLint zero warnings

### Layer 7: Pre-Commit / Pre-Push Hooks
Local CI equivalent. All validation runs at commit/push time, not in remote CI. This gives agents immediate feedback (seconds vs. minutes).

- **Pre-commit**: `.pre-commit-config.yaml` — lint, type checks, formatting
- **Pre-push**: Test suite execution
- **Post-push**: Codex reviews PRs at P0/P1/P2 severity tiers on GitHub

> [!WARNING]
> Pre-commit hooks only run on the machine that commits. If you create a worktree, ensure `pre-commit install` is run in it. The `--no-verify` flag bypasses all hooks.

**Rule of thumb**: RAG knows the *domain*. KIs know the *project*. MCP tools know the *runtime state*.

## 5. Gaps — What's Missing From the Augmentation Stack

### Gap 1: Auto-Refreshing Context (HIGH IMPACT)
**Problem**: KIs go stale between conversations. 6 weeks elapsed between Rosetta deletion and KI acknowledging it.
**Solution**: A git post-merge hook or periodic script that validates all KI `metadata.json` file references against the filesystem. Flag broken refs automatically.
**Effort**: ~50 lines of Python.

### Gap 2: API Surface Discovery (HIGH IMPACT)
**Problem**: Understanding available API endpoints requires 5+ tool calls (grep for routes, view schemas, etc.).
**Solution**: Auto-generate a slim API cheat sheet from `backend/scripts/export_openapi.py` output. Place in `docs/rag-knowledge/` for RAG indexing.
**Effort**: ~30 lines of Python to transform OpenAPI JSON → markdown summary.

### Gap 3: Persistent Failure Patterns (MEDIUM IMPACT)
**Problem**: When Agent A breaks imports by editing `block_assignment.py` without updating the import service, Agent B has no way to know this happened before. The lesson dies with the conversation.
**Solution**: A "known gotchas" registry that accumulates from recurring test failures. `docs/planning/DB_SCHEMA_REFERENCE.md` is a manual version of this pattern.
**Effort**: Requires cultural commitment more than tooling.

### Gap 4: Call Graph / Dependency Awareness (MEDIUM IMPACT)
**Problem**: Agents use text search (`grep`) to find callers/callees. This misses dynamic dispatch, Celery tasks, and frontend→API indirect calls.
**Solution**: Periodic `pyreverse` or `pyan3` export of the Python call graph, stored as a reference doc.
**Effort**: Script + periodic regeneration.

### Gap 5: Agent Telemetry (LOW IMPACT NOW, HIGH LATER)
**Problem**: No visibility into agent efficiency — tool calls per task, token usage, skill utilization.
**Solution**: Parse conversation logs from `~/.gemini/antigravity/brain/` into basic metrics.
**Effort**: ~100 lines of Python for initial dashboard.

## 6. Current CI/Quality Strategy

The project uses a **local-first** quality strategy rather than remote CI:

```
Agent codes
  → pre-commit hook (lint, types, format) — seconds
  → pre-push hook (tests) — minutes
  → push to GitHub
  → Codex reviews PR at P0/P1/P2 severity
  → Human merges after Codex approval
```

**Why this works**: Agents get immediate feedback from hooks (seconds) vs. waiting for CI (5-15 min). By the time code reaches GitHub, it's already been validated locally. Codex review adds a second LLM perspective before human review.

**Risk**: Hooks are local-only. Worktrees and `--no-verify` can bypass them. Running `pre-commit run --all-files` periodically catches accumulated drift.
