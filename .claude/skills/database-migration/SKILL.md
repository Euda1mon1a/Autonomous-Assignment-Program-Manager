***REMOVED*** Database Migration Skill

Expert guidance for safe database schema changes using Alembic migrations with SQLAlchemy models.

***REMOVED******REMOVED*** When This Skill Activates

- Adding new database models
- Modifying existing model fields
- Creating foreign key relationships
- Renaming columns or tables
- Dropping columns or tables
- Troubleshooting migration errors
- Rollback scenarios

***REMOVED******REMOVED*** Core Principle

**NEVER modify database models without creating a corresponding Alembic migration.**

***REMOVED******REMOVED*** Migration Patterns

***REMOVED******REMOVED******REMOVED*** Adding a Nullable Column (Safe)

```python
def upgrade() -> None:
    op.add_column('table_name',
        sa.Column('new_column', sa.String(), nullable=True)
    )

def downgrade() -> None:
    op.drop_column('table_name', 'new_column')
```

***REMOVED******REMOVED******REMOVED*** Adding a Non-Nullable Column (Requires Default)

```python
def upgrade() -> None:
    ***REMOVED*** Add as nullable first
    op.add_column('table_name',
        sa.Column('new_column', sa.String(), nullable=True)
    )
    ***REMOVED*** Backfill data
    op.execute("UPDATE table_name SET new_column = 'default_value' WHERE new_column IS NULL")
    ***REMOVED*** Make non-nullable
    op.alter_column('table_name', 'new_column', nullable=False)

def downgrade() -> None:
    op.drop_column('table_name', 'new_column')
```

***REMOVED******REMOVED******REMOVED*** Adding an Index

```python
def upgrade() -> None:
    op.create_index(
        'ix_table_column',
        'table_name',
        ['column_name']
    )

def downgrade() -> None:
    op.drop_index('ix_table_column', 'table_name')
```

***REMOVED******REMOVED******REMOVED*** Adding a Foreign Key

```python
def upgrade() -> None:
    op.add_column('child_table',
        sa.Column('parent_id', sa.String(), nullable=True)
    )
    op.create_foreign_key(
        'fk_child_parent',
        'child_table', 'parent_table',
        ['parent_id'], ['id'],
        ondelete='CASCADE'
    )

def downgrade() -> None:
    op.drop_constraint('fk_child_parent', 'child_table', type_='foreignkey')
    op.drop_column('child_table', 'parent_id')
```

***REMOVED******REMOVED******REMOVED*** Renaming a Column

```python
def upgrade() -> None:
    op.alter_column('table_name', 'old_name', new_column_name='new_name')

def downgrade() -> None:
    op.alter_column('table_name', 'new_name', new_column_name='old_name')
```

***REMOVED******REMOVED******REMOVED*** Dropping a Column (DANGER)

```python
def upgrade() -> None:
    ***REMOVED*** CAUTION: Data will be lost!
    op.drop_column('table_name', 'column_to_drop')

def downgrade() -> None:
    ***REMOVED*** Cannot restore data - only recreate column
    op.add_column('table_name',
        sa.Column('column_to_drop', sa.String(), nullable=True)
    )
```

***REMOVED******REMOVED******REMOVED*** Creating a New Table

```python
def upgrade() -> None:
    op.create_table(
        'new_table',
        sa.Column('id', sa.String(), primary_key=True),
        sa.Column('name', sa.String(), nullable=False),
        sa.Column('created_at', sa.DateTime(), server_default=sa.func.now()),
        sa.Column('updated_at', sa.DateTime(), onupdate=sa.func.now())
    )
    op.create_index('ix_new_table_name', 'new_table', ['name'])

def downgrade() -> None:
    op.drop_index('ix_new_table_name', 'new_table')
    op.drop_table('new_table')
```

***REMOVED******REMOVED******REMOVED*** Issue: Foreign Key Constraint Error

```python
***REMOVED*** Add column without FK first, then add constraint
def upgrade() -> None:
    ***REMOVED*** 1. Add column
    op.add_column('table', sa.Column('fk_id', sa.String()))

    ***REMOVED*** 2. Backfill valid values
    op.execute("""
        UPDATE table
        SET fk_id = (SELECT id FROM parent LIMIT 1)
        WHERE fk_id IS NULL
    """)

    ***REMOVED*** 3. Add constraint
    op.create_foreign_key('fk_name', 'table', 'parent', ['fk_id'], ['id'])
```

***REMOVED******REMOVED*** Escalation Rules

**ALWAYS escalate to human when:**

1. Dropping tables or columns with data
2. Changing primary keys
3. Large-scale data migration (>100k rows)
4. Production database migration
5. Modifying ACGME-related tables
6. Changes affecting audit trails
7. Unclear rollback path

**Can handle automatically:**

1. Adding nullable columns
2. Adding indexes
3. Creating new tables
4. Adding foreign keys to empty columns
5. Renaming columns (with verification)
