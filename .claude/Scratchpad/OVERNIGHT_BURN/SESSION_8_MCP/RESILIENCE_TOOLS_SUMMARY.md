## System Health Assessment

### Five Defense Levels (Escalation Model)

```
GREEN       ──── YELLOW      ──── ORANGE     ──── RED        ──── BLACK
Prevention       Control      Safety Sys    Containment    Emergency
Utilization      Monitoring  Automation     Zone Isolation  Full Fallback
<70%             70-80%       80-85%        85-95%          >95%
```

**Defense Level Responses**:
1. **GREEN**: Normal operations, focus on prevention
2. **YELLOW**: Enhanced monitoring, constraint tightening possible
3. **ORANGE**: Automated safeguards engaged, flexibility reduced
4. **RED**: Emergency zone isolation, blast radius limitation
5. **BLACK**: Full fallback activation, manual intervention required

## Emergency Response Workflow

### Crisis Detection & Resolution

```
1. DETECT (Automated)
   ├─ Utilization Threshold → RED/BLACK
   ├─ Defense Level → CONTAINMENT/EMERGENCY
   └─ Circuit Breaker → OPEN

2. ASSESS (Expert)
   ├─ run_contingency_analysis_resilience_tool()
   ├─ calculate_blast_radius_tool()
   ├─ get_unified_critical_index_tool()
   └─ get_static_fallbacks_tool()

3. ACTIVATE (Choose One)
   A. Fallback Schedule
      ├─ Select fallback scenario
      ├─ Verify ACGME compliance
      └─ Implement with audit trail

   B. Sacrifice Hierarchy
      ├─ Determine reduction target
      ├─ Execute triage decision
      └─ Communicate impact

4. MONITOR (Real-time)
   ├─ Circuit breaker health (5 min)
   ├─ Utilization trend (15 min)
   ├─ Allostatic load (hourly)
   └─ Coverage rate (continuous)

5. RECOVER (Systematic)
   ├─ Calculate recovery_distance to normal
   ├─ Plan reversal of sacrifice decisions
   ├─ Schedule gradual reintegration
   └─ Post-incident review (24 hr)
```
